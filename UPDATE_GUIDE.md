## 🚀 Быстрый старт

### Вариант 1: Обычный запуск (без Docker)

```bash
cd /opt/vpn_bot

# Активируйте виртуальное окружение
source venv/bin/activate

# Обновите базу данных
python -c "import database as db; db.init_db()"

# Запустите бота
python bot.py
```

### Вариант 2: Docker (рекомендуется)

```bash
# Установка Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Перейдите в директорию
cd /opt/vpn_bot

# Запустите
sudo docker compose up -d

# Смотрите логи
sudo docker compose logs -f bot
```

## 🔧 Настройка канала (обязательно)

### Шаг 1: Создание канала

1. Откройте Telegram
2. Меню → "Создать канал"
3. Тип: **Публичный**
4. Username: `@vpn_yourname` (придумайте)

### Шаг 2: Добавить бота

1. Настройки канала → Администраторы
2. "Добавить администратора"
3. Найдите бота по username
4. Дайте права на чтение сообщений

### Шаг 3: Получить ID канала

1. Добавьте [@userinfobot](https://t.me/userinfobot) в канал
2. Он покажет ID (например, `-1001234567890`)
3. Удалите @userinfobot

### Шаг 4: Обновите .env

```bash
nano .env
```

Добавьте:
```ini
CHANNEL_USERNAME=@vpn_yourname
CHANNEL_ID=-1001234567890
```

### Шаг 5: Перезапустите бота

```bash
# Обычный запуск
sudo systemctl restart vpnbot

# Docker
sudo docker compose restart bot
```

## 📊 Статистика трафика — как работает

### Получение данных

Бот получает данные из 3x-ui панели через API:
- `upload` — загружено клиентом (ГБ)
- `download` — скачано клиентом (ГБ)
- `total` — всего использовано (ГБ)
- `limit` — лимит трафика (если установлен)
- `remaining` — осталось до лимита

### Отображение

Пользователь нажимает "📊 Статистика" → видит:
```
📊 Статистика трафика

📤 Загружено: 1.25 ГБ
📥 Скачано: 5.67 ГБ
📈 Всего: 6.92 ГБ

📉 Лимит: 100 ГБ
⏳ Осталось: 93.08 ГБ

[█████░░░░░░░░░░░░░░] 6.9%
```

### Админ-команды

```bash

# Посмотреть весь трафик
```

## 🐳 Docker — подробная инструкция

### Структура

```
vpn_bot/
├── docker-compose.yml    # Конфигурация сервисов
├── Dockerfile            # Образ бота
├── nginx.conf            # Настройка nginx
└── .dockerignore         # Исключения для Docker
```

### Сервисы

1. **bot** — Python бот (порт 8080)
2. **nginx** — Reverse proxy (порты 80, 443)
3. **certbot** — SSL сертификаты

### Команды

```bash
# Запуск
sudo docker compose up -d

# Логи
sudo docker compose logs -f

# Перезапуск
sudo docker compose restart

# Остановка
sudo docker compose down

# Обновление
sudo docker compose pull
sudo docker compose up -d
```

### SSL сертификаты

```bash
# Получить сертификат
sudo docker run --rm -it \
    -v /opt/vpn_bot/ssl:/etc/letsencrypt \
    certbot/certbot certonly --standalone \
    -d your-domain.com \
    --email your@email.com

# Обновить
sudo docker compose run --rm certbot renew
```

## 🐛 Решение проблем

### Бот не проверяет подписку

1. Проверьте `CHANNEL_ID` в `.env`
2. Убедитесь что бот — администратор канала
3. Перезапустите бота

### Статистика трафика не работает

1. Проверьте подключение к 3x-ui
2. Убедитесь что `XUI_INBOUND_ID` верный
3. Проверьте логи: `grep "traffic" logs/app.log`

### Docker не запускается

```bash
# Проверить конфиг
sudo docker compose config

# Проверить логи
sudo docker compose logs bot
sudo docker compose logs nginx
```

### Ошибка "port already in use"

```bash
# Найти процесс
sudo lsof -i :8080

# Остановить или изменить порт
```
